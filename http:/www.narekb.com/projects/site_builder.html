Static Site Generator

I decided to build a static website generator. I did it to simplify the process of posting my projects online, because I'd rather produce content than get wrapped up in web development.

I tried other static site builders but ran in to npm issues that I didn't feel like dealing with. I tried to keep this project simple, with the only frivolous addition being an RSS feed generator, since building projects should be the main goal still.

GitHub Link

Scope

Objective

Build a static website generator that allows me to focus on writing build logs and working on projects instead of web development.

Requirements

  • Build website from markdown files
  • Generate RSS feeds at build time at baseurl/rss
  • Easy use & extensibility

Constraints

  • Minimal modification of markdown
  • No complicated web features (hidden menus, etc..)
  • Minimal web design overall
  • Only markdown files will be used for text input

Features

  • Markdown processing
  • Template based website construction
  • RSS Feed
  • Home page generator
  • Navigation breadcrumbs
  • Sort entries based on date
  • Auto generated category pages
  • Contact form
  • Include month, year below title within articles
  • Automated bibliography and References
  • Table of contents generator
  • Tags
  • Links for sections
  • Search
  • LaTex Math integration
  • Input file format versioning

Design

The Tech

I decided to use python and markdown for their simplicity and faster development time. I considered creating a file format but markdown has almost everything I want already. The only thing missing from markdown is a native way to add metadata. To circumvent this, I decided to add a metadata section before the first '#' and pre-process the markdown to consume and remove this section before converting it to HTML using the markdown package in python.

Processing Files

The input is configurable. Here src and build directories are git submodules so the content can be updated independently from one another. Python searches recursively through the src directory where files are organized in a hierarchy that corresponds to their final url. For example:

src
└── projects
    └── site-builder.md

becomes

<site>/projects/site-builder/index.html

but is also accessible through

<site>/projects/site-builder

Those files are filtered to determine which files are valid markdown files, the contents and metadata are separated, the contents are converted to HTML while the metadata is processed.

Templating

A template is used to generate the HTML. The idea here is that templates can be configured for the specific function and selected in the file metadata. Templates are stored as html files with some pattern strings that will be matched to, such as {title}.

Generating RSS

The metadata and some extracted content is used to generate the RSS feed. This feed is updated every time the website is re-built and is accessible at /rss/rss.xml .

Home Page Generator

This generates based on a home page specific template. In doing this, I had to introduce another metadata variable 'type' which is unnecessary and redundant. This whole project would be a lot better if it was done using object oriented programming, but this is one of those projects that changed as I did it. Oh well ¯\_(ツ)_/¯

Breadcrumbs

Breadcrumbs are the simple navigation elements at the top of this page. This is a list which has had some css added to make them linear and links have been added provide navigation. This is generated by iterating through filenames to generate links, and searching through the metadata to find the title name. Once this is done, a breadcrumb template is formatted and the html is added to the site, via the primary template.
