Imagemagick

Note - Imagemagick is for images, not vectors. It will convert fine but native pdf format will balloon in size. Can maintain jpg format so size doesn't balloon. Or use ghostscript

Resolution issues

'-density X' will fix some issues with resolution where '-quality 100' will not

Last I used for X was 400 (-density 400)

Convert Images to PDF

magick "IMG"* out.pdf

Change size

To Letter

convert in.pdf -page letter out.pdf

From Images

convert *.jpg -units pixelsperinch -density 72 -page letter foo.pdf

To specific pixel width, maintaining aspect

-resize 100x

(Geometry in general is x )

Background Color

convert image.tif -resize 575x823^> -gravity center -background white -extent 595x842 image.pdf

Change DPI

convert -density 200x200 -quality 60 -compress jpeg input.pdf output.pdf

(For JPEG, obviously)

convert -units PixelsPerInch input.png -density 300 output.png

(This one sucks. Size balloons. See top note)

Convert text to png

  • https://unix.stackexchange.com/questions/138804/how-to-transform-a-text-file-into-a-picture
  • https://legacy.imagemagick.org/discourse-server/viewtopic.php?t=18300
  • https://askubuntu.com/questions/673615/imagemagick-convert-command-cannot-use-fonts
convert -size 1200x140 xc:"rgba(0,0,0,0)" -type truecolormatte -font "Helvetica-Neue-Bold" -pointsize 120 -fill blue -draw @email.txt PNG32:email.png

where email.txt is...

text 48,100 "email@domain.tld"

represents the offset of the text, and it is stored in the txt itself…

convert -list font | grep "Font:"

can use this to find all available fonts…
